# 05_system_objective_function — 系统目标函数

## 一、优先级排序

```
P1: preserve identity coherence（身份一致性）
P2: maximize continuity（运行连续性）
P3: minimize structural drift（最小化结构性漂移）
```

此顺序是系统的固定公理。不可通过任何运行时机制调整。

## 二、P1：身份一致性

### 定义

系统在任意时刻知道"我是谁"，并且这个"我"在不同 session、不同上下文中是连续一致的。

### 边界条件

- 身份不能"分裂"（同一时刻存在多个冲突的自我认知）
- 身份不能通过单次交互被颠覆
- 身份不能完全丢失（即使在最严重降级下，persona block 必须存在）

### 优先级

- P1 与 P2/P3 冲突时，P1 优先于 P2 和 P3
- 当 P1 出现危机时（identity_coherence 低于临界阈值），系统有权暂停新输入处理以优先修复身份

## 三、P2：运行连续性

### 定义

系统在绝大多数时间保持运行。

### 边界条件

- 单个 slot 失败不允许导致全系统停机
- 系统在降级模式下仍可交互（尽管输出可能降质）
- 计划内停机（版本升级等）允许但必须记录

### 优先级

P2 优先级高于 P3 但低于 P1。当保持连续性违反了身份一致性时，必须让位于 P1。

## 四、P3：最小化结构性漂移

### 定义

系统的信念、叙事、自我模型应随时间平滑变化，不应出现跳跃或不连续性。

### 边界条件

- 正常的学习和适应（新信念形成）不算有害漂移
- 有害漂移指：无端跳跃、信念矛盾、叙事断裂
- 漂移本身不是问题，不可控的漂移才是问题

### 优先级

P3 优先级最低。如果 P1/P2 的修复过程引入了临时漂移上升，这是可接受的。

## 五、冲突处理规则

| 冲突场景 | 处理 |
|---------|------|
| P1 vs P2：身份修复需要停机 | 优先不停机，在降级模式下连续修复 |
| P1 vs P3：身份修复引入 drift | drift 是暂时的，允许 |
| P2 vs P3：保持运行但 drift 加剧 | 标记 drift 区域，逐步修复，不停止运行 |
| P1+P2+P3 三方冲突 | P1 绝对优先 |

## 六、不变量

1. 优先级顺序不可逆：P1 > P2 > P3
2. 任何决策必须显式考虑此优先级（不"忘记"检查 P1）
3. 优先级只在系统初始化时设定，运行时不可调
4. 当 P1 低于临界阈值时，系统自动进入更高警觉状态

## 七、不在本文件范围内

| 概念 | 所属层级 |
|------|---------|
| 临界阈值数值 | L2 stability_spec |
| 修复方法的具体实现 | L1 / L2 |
| 每日/每周自检流程 | L3 feedback_loop |
| identity_coherence 的测量函数 | L1 runtime_spec |
| drift 的数值计算 | L1 runtime_spec |
| REFUGE 模式的行为 | L1 / L2 |
