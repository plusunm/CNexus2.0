# 02_minimal_cognitive_loop — 最小认知回路

## 一、定义

这是系统认知的基本结构。回路定义系统如何从输入到输出的完整路径。

回路不可再拆分：移除任何一步，系统就不再是一个认知系统。

## 二、回路结构

```
OBSERVE → COGNIZE → DECIDE → SPEAK → STORE → REFLECT
                                                       ↓
                                            （回到 OBSERVE）
```

| 步 | 名称 | 定义 |
|----|------|------|
| OBSERVE | 感知 | 系统接收外部或内部输入，并将其转换为标准化的内部表示 |
| COGNIZE | 认知处理 | 系统结合当前状态和历史记忆理解输入，生成认知上下文 |
| DECIDE | 决策 | 系统根据认知上下文选择输出策略 |
| SPEAK | 输出 | 系统执行选定的输出策略，生成自然语言或其他形式的响应 |
| STORE | 存储 | 系统将本轮交互的完整记录写入记忆 |
| REFLECT | 反思 | 系统评估本轮交互的影响，调整自身状态 |

## 三、不变量

### 3.1 结构不变量

1. **回路不可拆分：** 6 步结构不可压缩或合并
2. **步序不可逆：** OBSERVE 不能发生在 COGNIZE 之后
3. **OBSERVE + STORE 必须存在：** 即使在最严重的降级下，感知和存储的路径也必须保留
4. **REFLECT 可以缺失：** 反思不影响本轮输出的及时性

### 3.2 行为不变量

1. **系统必须推进状态：** 每轮循环结束时，系统状态必须比开始时有所不同（至少计数器增加）
2. **系统不因单步失败而停止整体：** 任何一步的失败应允许下一步继续执行
3. **系统永远在循环中：** 主循环不退出（除非关机）

### 3.3 边界条件

1. 单轮循环的输出不能作为同一轮循环的输入（防止自反馈循环）
2. 在无输入状态下（IDLE），系统定期执行 OBSERVE 检查新输入，但不执行完整循环
